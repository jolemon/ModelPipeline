设计 — 项目结构 & 组件

目录结构

model_report/ ├── __init__.py # 公开 API: ReportGenerator, ScorecardProtocol, load_metadata ├── interface.py # ScorecardProtocol 抽象接口定义 ├── config.py # ReportConfig: 列名映射 + 标签 + 阈值等可配置项 ├── metadata.py # 变量清单加载 (CSV/YAML → vardict) ├── generator.py # ReportGenerator 编排器 ├── cli.py # CLI 入口 (argparse/click) ├── writer.py # ExcelWriter: openpyxl 写入 + 条件格式 (数据条) ├── sheets/ │ ├── __init__.py │ ├── model_design.py # Sheet 1: 模型设计 — 分区分布 + 建模分 │ ├── variable_analysis.py # Sheet 2: 变量分析 — IV/KS/PSI + Top10 WOE │ └── model_performance.py # Sheet 3: 模型表现 — 评分卡详情 + 效果 + 回溯 + 分箱

核心组件职责

ScorecardProtocol (interface.py)
抽象接口，定义报告生成器需要的评分卡信息。不依赖 scorecard_jsb.py。
提供两个实现：BinnerScorecardAdapter（适配现有对象）和 PickledScorecardAdapter（从 .pkl 加载）。
必需属性： bins, woetables, ivtable, model_result (系数/Wald), scorecard (分箱→分映射), dropped_vars
必需方法： get_var_bins(var) → DataFrame, get_model_summary() → DataFrame
ReportConfig (config.py)
dataclass，所有可配置项有默认值（与 readme 规格一致）。
列名映射: partition_col="part_id", cust_col="cert_no", date_col="loan_date", target_col="mob6_30", flag_col="data_flag", score_col="pred_score", sc_score_col="scorecard_score"
标签: partition_labels, target_label="Mob6 30+", sheet_names
阈值: bin_count=20, top_n_vars=10
Sheet 生成器 (sheets/*.py)
每个 sheet 模块暴露一个 build_*_sheet(data, scorecard, config, metadata) → list[dict[str, DataFrame]] 的函数。
返回结构化数据（DataFrame 列表），不含任何 Excel 写入逻辑。
计算与格式化完全分离。
ExcelWriter (writer.py)
接收 sheet 生成器产出的结构化数据，写入 Excel。
负责：表头样式（粗体、背景色）、数字格式（百分比、小数位）、条件格式（数据条、实心填充）、列宽自适应。

数据流

DataFrame + ScorecardProtocol + ReportConfig
↓
ReportGenerator.generate()
↓
内部调用 3 个 sheet builder → 得到结构化数据
↓
ExcelWriter.write(workbook, structured_data)
↓
output.xlsx
