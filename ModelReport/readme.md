# 模型报告生成器

# 背景
这是一个模型报告生成器，具体指信贷风控领域的模型。当前我们已使用LR模型建模。

# 输入
模型报告生成器的输入一般是
1、模型文件：LR模型是.pkl格式评分卡
2、打分结果（pandas.DataFrame），包含以下列：
- 样本分区(partition_key)：列名默认为“part_id”，取值格式如“yyyyMM”。
- 样本客户主键(cust_key)：列名默认为“cert_no”，一般为md5加密的格式，与样本时间作为联合主键保证全局唯一。
- 样本时间(sample_date)：列名默认为“loan_date”，样本发生时间，用于向前回溯变量。取值格式如“yyyy-MM-dd”。
- 样本标签（target_label）：列名默认为“mob6_30”，“mob6_30”意为“在6个账龄月内是否发生逾期30天以上”，0表示未逾期，1表示逾期。
- 样本数据集划分标签(data_flag)：列名默认为“data_flag”，取值为train/test/oot/oos，分别代表训练集（必含）、测试集（必含）、跨时间验证集（必含，时间外样本测试）、压测集（表现期未满标签要求的样本测试，可以不包含）。
- 样本分数（pred_score）：：列名默认为“pred_score”，一般为0-1之间的分数，如逻辑回归模型的输出值
- 评分卡分数（scorecard_score）：列名默认为“scorecard_score”，从评分卡映射后的分数，为正数，一般精度为0-2位小数。
- 样本所有变量（多列）：列名如feat1、feat2、…，均为数值型。
- 其他备用列（不包含在变量列中，且未在上述声明配置的剩余字段，不需关注）

# 输出
一个excel格式的模型报告，包含多个sheet。
1、“模型设计”sheet
1）打印样本分区分布
列：样本数据集划分标签、样本分区、好、坏、总数、坏占比（=坏/总数，格式如1.49%，保留小数点后2位）
行：按样本分区顺序排列，最后是“总计”行。
2）打印样本建模分
列：样本数据集划分标签、好、坏、总数、坏占比（=坏/总数，格式如1.49%，保留小数点后2位）
行: 固定4行，分别为：训练集（train）/测试集(test)/跨时间验证集(oot)/“总计”。
